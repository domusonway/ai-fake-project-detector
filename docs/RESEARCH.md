## 调研目标
- 理解“莫名其妙爆火但难以判断价值的 AI 项目”这一用户痛点

- 识别同类产品、替代方案和潜在数据来源

## 核心洞察
- 事实：用户需要一个专门面向“爆火但难判断价值”的 AI 项目鉴别器
- 事实：用户期待的不只是单点分析，而是“结构分析 + 同类对比 + 评分 + 排名 + 投票”组合产品
- 推断：该产品的核心差异化不在于发现热门项目，而在于提供可解释的 fake risk 判断
- 推断：仅靠 GitHub stars 或 Twitter/X 热度不足以判断项目是否 fake，必须结合结构和可运行性特征
- 假设：用户愿意参与投票，但前提是评分解释足够透明，且投票不会明显被刷
- 假设：同类项目对比会显著提升评分说服力，因为单项目评分容易引发争议
- 待验证：fake project 的定义边界是否应覆盖“纯文档项目”“营销驱动项目”“Demo 不可运行项目”“二次包装项目”
- 待验证：公开排名是否会带来法律、声誉或社区治理风险
